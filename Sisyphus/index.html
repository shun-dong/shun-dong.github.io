Claude code

[[Anthropic]] 是一家有宗教氛围的公司, 其先后提出 [[MCP]], [[AI#skill]] 等工具使用规范, 并发布 Claude code 这一 CLI(command line interface, 命令行交互), 并形成了具有相当规模的生态系统.
shareAI-lab/learn-claude-code: Bash is all you need - A nano Claude Code–like agent, built from 0 to 1 该项目对于理解其原理有极大帮助.

尽管本身技术并不复杂, 且有众多同类的开源项目, 如 codex, gemini cli, [[opencode]], 但 Claude code 从未真正开源, 且该公司一定程度上站在开源者的对立面. 在2026年3月31日, 因为 npm 打包时泄露 source map, 源代码被逆向在 [[github]] 传播, 但该公司以 [[DMCA]] 为由下架了大量开源项目.

paoloanzn (Paolo Anzani) 在2026年3月31日泄露的源码的基础上打开了一些测试功能并禁用了该公司的审查, 我在此基础上打开了 auto-mode 的功能并微调, 最终呈现为 [[fcode]]. 目前准备长期使用 fcode, 专注于实践, 不囿于形式.

  1. 现在范式趋于稳定, 更新版本反而可能会遇到不兼容的问题
  2. 自主可控, 安全
  3. 对于此版本的cc, 网上有大量资料

opencode

以开源和美观著称的 AI cli 工具, 有著名的社区插件 oh-my-opencode, 通过主 agent 调度子 agent 以及一套完整的流程实现了长时间自主运行的能力, 但也有工作流过于冗杂带来的高消耗和混乱问题.

由于 AI 工具的发展, 众多开发者提出 issue 和 pull request, 远超出项目组的审查能力, 导致诸如 todo read 工具被无通知地禁用的问题, 实际上阻碍了这个项目的发展.
